NAI Image Manager v1.0.0 Lite Windows 배포

날짜: 2026-06-07
태그: v1.0.0-lite
브랜치: codex/lite-exe-packaging
릴리즈: GitHub Releases

요약

Windows에서 바로 실행할 수 있는 Lite 배포판을 추가했습니다. 기존 대용량 빌드에서 torch, CUDA, transformers 계열 AI 자동분류 패키지를 제외하고, 기본 갤러리/캔버스 기능과 Real-CUGAN 업스케일 도구 중심으로 구성했습니다.

배포 zip에는 개인 설정 파일과 캔버스 저장본을 포함하지 않도록 정리했으며, 소스코드로 실행하는 기존 사용자의 동작은 유지되도록 exe 전용 조건으로 분리했습니다.

주요 변경 사항

Lite exe 패키징 추가

NAI_Image_Manager.spec를 추가해 PyInstaller 기반 Windows 배포본을 만들 수 있게 했습니다. 실행 파일은 같은 exe를 --server 모드로 다시 실행해 Flask 서버를 띄웁니다.

용량 축소

AI 자동분류용 대형 패키지를 제외해 릴리즈 zip을 약 96MB 수준으로 줄였습니다. Real-CUGAN 업스케일러와 모델 파일은 그대로 포함됩니다.

AI 자동분류 제한

Lite exe에서는 AI 자동분류 패키지가 포함되지 않으므로 AI 체크박스와 설치 API를 제한합니다. 소스 실행에서는 기존 AI 설정과 설치 흐름을 유지합니다.

개인 설정 제외

배포 zip에서 갤러리 설정, 스타일 설정, 캔버스 저장본 같은 로컬 데이터 파일을 제외했습니다. 첫 실행 후 필요한 설정 파일은 사용자 환경에서 새로 생성됩니다.

캔버스 저장본 자동 복구 차단

Lite exe에서는 브라우저 localStorage에 남아 있던 기존 캔버스 저장본을 새 배포본으로 자동 마이그레이션하지 않도록 했습니다.

exe 기준 경로 보정

frozen exe에서는 실행 파일 폴더를 기준으로 정적 파일, DB, 업스케일 도구, 작업 폴더를 찾도록 보정했습니다.

포함된 항목

  • NAI_Image_Manager.exe
  • 갤러리, 캔버스, 스타일 연구소, 다키 워크숍 화면
  • Real-CUGAN 업스케일 실행 파일과 모델 파일
  • Danbooru 태그 DB 및 기본 태그 데이터
  • Flask, Pillow 등 앱 실행에 필요한 기본 런타임 의존성

제외된 항목

  • torch, CUDA, transformers, tokenizers 기반 AI 자동분류 패키지
  • gallery_config.json
  • lab_config.json
  • artists.json, styles.json
  • gallery_image_tags.json
  • tag_dictionary_user_overrides.json
  • canvas_saved_setups.json, quality_presets.json
  • 사용자 이미지 라이브러리와 생성 결과 폴더

사용 방법

  1. GitHub Release에서 NAI_Image_Manager_v1.0.0-lite_windows.zip을 다운로드합니다.
  2. 원하는 위치에 압축을 풉니다.
  3. 압축 해제된 폴더 안의 NAI_Image_Manager.exe를 실행합니다.
  4. exe 파일만 따로 옮기지 말고, 압축 해제된 폴더 전체 구조를 유지합니다.

검증 내용

  • python -m py_compile로 주요 Python 파일 문법 검사 통과
  • python -m PyInstaller --clean --distpath dist_lite 빌드 성공
  • zip 내부 개인 설정 파일 검사 결과: NO_PRIVATE_CONFIGS_IN_ZIP
  • AI 대형 패키지 파일 검색 결과: 0
  • 업스케일러, HTML, 태그 DB 포함 여부 확인
  • GitHub Release asset 교체 및 다운로드 URL 확인

참고

이 패치노트는 별도 JavaScript 없이 HTML 태그와 <style>만 사용해 작성했습니다. Lite exe는 배포 편의를 위한 별도 산출물이며, 소스코드로 실행하는 기존 사용자의 AI 및 캔버스 저장본 동작은 유지됩니다.
