# v1.1.0 EXE Patch Notes HTML Design

## Goal

Create `patch_notes/2026-06-20-v1.1.0-exe-release.html` as a standalone Korean patch note for the published `v1.1.0-exe` release.

## Visual Direction

Reuse the existing v1.0.0 Lite patch note's dark background, bordered cards, colored left accents, compact badges, and Malgun Gothic typography. All presentation must use HTML tags and inline `style` attributes only. Do not add JavaScript, `<script>`, external stylesheets, `<style>` blocks, or separate CSS files.

## Content

The page will include release metadata and a GitHub Release link, an executive summary, EXE self-update behavior, legacy Lite migration steps, user-data preservation guarantees, gallery prompt search indexing, source-versus-EXE branch policy, installation instructions, Lite limitations, verification evidence, SHA-256, and a concise modified-file list.

## Safety and Accuracy

The HTML must use UTF-8 Korean text, escape code-like filenames safely, link only to the published GitHub Release, and match the final release tag, asset name, test count, managed-file count, and SHA-256 already recorded in the Markdown release notes.

## Verification

Check that the file contains no `<script>`, `<style>`, stylesheet link, or JavaScript URL; parse its links and key metadata; and render it locally for a visual inspection of Korean text, spacing, card borders, and line wrapping.
